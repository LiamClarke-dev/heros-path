# Map Pins Feature Brief

## Overview

This feature adds saved-place pins to the map experience in two phases. The first phase is scoped to individual lists — when a user taps into a saved list, they see a map showing all the places in that list. The second phase brings optional pin overlays to the main journey map screen, controlled by a filter.

## Phase 1 — List Map Card

When a user navigates to a saved list, a map card should appear at the top of the list view, pinned in place so it's always visible. This card renders pins for every location saved to that list. Tapping the map card expands it into a fullscreen interactive map. This keeps the spatial context immediately visible without requiring an extra navigation step — the user opens a list and instantly sees where their saved places are relative to each other.

## Phase 2 — Journey Map Pin Filter

On the main journey/explore map screen, add a filter control that lets the user overlay pins from their saved lists. The default state should be "none" — no saved-place pins shown — so the map stays clean and focused on discovery. The filter should use a bottom sheet with toggleable chips, one per list, each showing the list's emoji. This lets users overlay multiple lists simultaneously (e.g., coffee spots and date night places at the same time) rather than being locked to a single list. An "All" toggle at the top handles the show-everything case.

## Pin Tap Behavior

When a user taps a pin, show a compact callout card that slides up from the bottom of the map — not the native map callout bubble, which is difficult to style and looks inconsistent. The card should show the place name, primary type (café, bar, park, etc.), and the photo if available. Two actions from the card: "View Details" (navigates to the place detail or opens the visit log sheet) and "Directions" (deep links to Apple Maps or Google Maps). One tap to see info, two taps to act.

On the journey map with multiple lists overlaid, the callout should also display a small tag indicating which list the pin belongs to, so the user understands the context for why that pin is on the map.

## Pin Visual Design

Each pin should use the list's emoji as its marker, rendered inside a small styled circle or teardrop shape with a subtle drop shadow. Since every list already has a user-chosen emoji, this makes pins instantly scannable and personally meaningful — ☕ for the coffee list, 🌹 for date night, etc. This approach is more accessible than color-coding alone (which fails for colorblind users) and more intuitive than numbered or letter-coded pins. A muted background tint per list can serve as secondary reinforcement, but the emoji should do the heavy lifting.

## Clustering

When multiple saved places are close together, pins will overlap and become unreadable at lower zoom levels. Use marker clustering to group nearby pins into a single circle with a count badge (e.g., "4") that expands into individual pins as the user zooms in. This is especially important for phase 2 where multiple lists can be overlaid at once. `react-native-maps` supports clustering natively.
